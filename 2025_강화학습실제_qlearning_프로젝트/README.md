# Q-Learning 기반 GridWorld 동적 경로 탐색

> 2025 강화학습 실제 강의 중간 프로젝트 — Q-learning 알고리즘을 이용해 GridWorld 환경에서 최적 경로를 학습하고, 동적 장애물 변화에 대응하여 경로를 재탐색하는 에이전트를 구현함.

## 프로젝트 개요

- 강화학습의 대표 알고리즘인 **Q-learning** 을 처음부터 직접 구현하여, 격자 환경(GridWorld)에서 시작점부터 목표점까지의 최적 경로를 학습시킴.
- 학습이 완료된 이후에도 환경에 **동적 장애물(dynamic walls)** 이 추가되었을 때, 학습된 Q-table을 활용하여 우회 경로를 자동으로 탐색하도록 확장함.
- Q-value, V-value, Policy, 경로를 격자 위에 시각화하여 학습 과정과 결과를 직관적으로 분석할 수 있도록 함.

## 환경 설계

- 4×4 크기의 `FourGridWorld` 환경을 정의함.
- 시작 상태는 `(0, 0)`, 목표 상태는 `(3, 3)`, 정적 장애물은 `(1, 1), (1, 3), (2, 3), (3, 0)` 에 위치함.
- 행동 공간은 `상(0), 하(1), 좌(2), 우(3)` 4가지로 구성됨.
- 보상 체계는 다음과 같이 설계함.
  - 목표 도달: **+1**
  - 장애물 진입: **-1**
  - 일반 이동: **-0.01** (불필요한 탐색을 줄이기 위한 step penalty)
- 한 에피소드 당 최대 스텝 수는 100으로 제한함.

## Q-Learning 에이전트

- 행동 정책으로 **ε-greedy** 를 사용하며, 학습이 진행됨에 따라 ε 값을 지수적으로 감소시켜 탐험(exploration) → 활용(exploitation) 으로 자연스럽게 전환되도록 함.
- 하이퍼파라미터:
  - 할인율 γ = 0.9
  - 학습률 α = 0.8
  - ε 초기값 = 1.0, 최소값 = 0.1, decay = 0.0001
  - 학습 에피소드 = 10,000
- Q-value 업데이트는 표준 Q-learning 업데이트 식을 따름.

  ```
  Q(s, a) ← Q(s, a) + α · [ r + γ · max Q(s', a') − Q(s, a) ]
  ```

## 동적 장애물 대응

- 학습이 끝난 후 `add_dynamic_wall(position)` 메서드를 통해 임의의 위치에 장애물을 추가할 수 있도록 함.
- `find_path_with_dynamic_walls(Q)` 함수에서 학습된 Q-table을 기반으로, 현재 상태에서 이동 가능한 행동 중 Q-value가 높은 순으로 후보 행동을 정렬한 뒤, 이미 방문한 상태나 막힌 경로는 제외하고 차선의 경로를 선택하도록 구현함.
- 다음 두 가지 케이스에 대해 우회 경로 탐색 동작을 검증함.
  - **Case 1**: `(0,0) → (0,1)` 경로가 막힌 경우
  - **Case 2**: `(2,2) → (3,2)` 경로가 막힌 경우

## 시각화

- `gridworld_render.py` 모듈을 통해 다음 요소를 격자 위에 시각화함.
  - 각 상태의 V-value (color map)
  - 각 상태-행동 쌍의 Q-value (삼각형 분할 color map)
  - 최적 정책(policy)
  - 시작점부터 목표점까지의 경로 (선 + 점 마커)
  - 정적 장애물(회색)과 동적 장애물(빨간색 반투명)
- 학습 추이 그래프로 **Steps per Episode**, **Reward per Episode**, **Epsilon Decay** 를 함께 출력하여 학습 수렴 과정을 분석함.

## 실행 방법

```bash
python q_learning.py
```

- 실행 시 다음 순서로 동작함.
  1. 기본 그리드월드 시각화 출력
  2. 10,000 에피소드 Q-learning 학습 수행
  3. 학습 추이 그래프(스텝/보상/입실론) 출력
  4. 학습된 Q-table과 V-value, Policy 시각화
  5. 기본 경로 탐색 결과 출력
  6. Case 1, Case 2 동적 장애물 시나리오에서 우회 경로 탐색 결과 출력

## 배운 점

- 단순한 격자 환경에서도 보상 설계, 학습률, ε-decay 스케줄링이 학습 수렴 속도와 안정성에 큰 영향을 줌을 확인함.
- 학습된 Q-table은 단순한 정책 추출뿐 아니라, **환경 변화에 대한 차선책 탐색의 근거**로도 활용될 수 있음을 직접 구현해보며 체감함.
- 강화학습에서 시각화는 디버깅과 결과 해석에 있어 매우 중요한 도구임을 확인함.
